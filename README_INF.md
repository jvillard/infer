## Capture

Only needed once.

version used: 5.19.1

```
make allyesconfig
infer capture \
   --compilation-database compile_commands.json \
   --keep-going \
   --clang-block-listed-flags -fasan-shadow-offset=0xdffffc0000000000 \
   --clang-block-listed-flags -fconserve-stack \
   --clang-block-listed-flags -fno-allow-store-data-races \
   --clang-block-listed-flags -fno-code-hoisting \
   --clang-block-listed-flags -fno-conserve-stack \
   --clang-block-listed-flags -fno-ipa-cp-clone \
   --clang-block-listed-flags -fno-partial-inlining \
   --clang-block-listed-flags -fno-tree-loop-im \

   --clang-block-listed-flags -mindirect-branch-register \
   --clang-block-listed-flags -mindirect-branch=thunk-extern \
   --clang-block-listed-flags -mindirect-branch=thunk-inline \

```

## Analysis and results

```
infer --pulse-only \
      --pulse-experimental-infinite-loop-checker-v2 -F \
      @syscalls_and_drivers.regex

```


The results can be analysed in various ways, for example using [`jq`](https://jqlang.org/).

- get all INFINITE_LOOP bugs reachable from entry points:

```
jq 'map(select((.extras.reachable_from | length > 0) and .bug_type == "INFINITE_LOOP"))' < infer-out/report.json
```

- get all *unique* INFINITE_LOOP bugs reachable from entry points:

```
jq 'map(select((.extras.reachable_from | length > 0) and .bug_type == "INFINITE_LOOP")) | map(.bug_trace[-1]) | unique_by(.filename, .line_number)' < infer-out/report.json
```
